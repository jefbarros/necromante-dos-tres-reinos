# G3.0 - MVP Godot jogavel completo

## Objetivo

Fechar um vertical slice jogavel e apresentavel em Godot. A G3.0 nao tenta criar um jogo completo de grande escala; ela entrega uma run curta com inicio, combate, ondas, necromancia minima, servo, progressao, inventario simples, vitoria/derrota e export Web.

Godot segue como plataforma principal. O HTML5/Canvas v0.3.4 segue congelado como legado e laboratorio de referencia.

## Sistemas implementados

- Estados de jogo: start, playing, victory e game over.
- Reinicio da run sem recarregar a pagina.
- Movimento do player com WASD/setas.
- Vida, mana, ataque basico, cooldown e feedback visual.
- Inimigos perseguem o jogador e causam dano por proximidade.
- Tres tipos de inimigo: carcaca veloz, devorador e guardiao da cripta.
- Sistema simples de ondas com vitoria ao limpar a ultima onda.
- Almas/corpos gerados ao derrotar inimigos.
- Captura/ressurreicao com servo basico.
- Limite de 3 servos.
- Servos seguem o jogador e atacam inimigos proximos.
- XP, level up, aumento simples de vida/mana/dano e restauracao parcial.
- Drops simples: pocao de vida, essencia de alma e fragmento sombrio.
- Inventario basico no HUD com painel alternavel.
- Tela de vitoria e tela de derrota.
- HUD completo para o MVP.

## Controles

- `Enter`: iniciar ou reiniciar.
- `WASD` ou setas: mover.
- `1` ou clique: ataque basico.
- `C`: capturar alma e erguer servo proximo.
- `I`: abrir/fechar inventario.
- `Q`: usar pocao de vida, se houver.
- `R`: reiniciar quando a run nao esta em jogo.

## Como testar localmente

1. Abrir o projeto em `godot/`.
2. Rodar `res://scenes/Main.tscn`.
3. Pressionar `Enter`.
4. Derrotar inimigos da primeira onda.
5. Aproximar-se de uma alma e pressionar `C`.
6. Confirmar que o servo segue e ataca.
7. Avancar ate a onda do guardiao.
8. Confirmar tela de vitoria ao limpar a ultima onda.
9. Reiniciar pela tela final.

## Como exportar Web

```bash
godot/Godot_v4.6.2-stable_win64.exe --headless --path godot --export-release "Web" ../docs/play-godot/index.html
```

Se `godot_console` estiver no PATH, ele tambem pode ser usado com os mesmos argumentos.

## Responsividade

O MVP G3.0 foi validado como alvo principal para desktop e notebook.

Mobile Web e experimental. Mobile horizontal e a orientacao recomendada para testes rapidos; mobile vertical pode carregar, mas tem limitacoes de espaco e ergonomia. A etapa G3.0.1 documenta os ajustes de responsividade, os tamanhos testados e as limitacoes conhecidas.

Ver `docs/godot/G3.0.1_RESPONSIVE_WEB.md`.

## Bugs conhecidos

- Balanceamento ainda e inicial e pode variar bastante entre runs.
- Arte ainda e procedural simples.
- O inventario nao possui tela de detalhes ou equipamentos.
- Servos nao possuem evolucao propria.
- O teste em GitHub Pages ainda precisa ser feito apos merge/publicacao.
