# G2.1 - Baseline visual e jogavel minimo

## Objetivo da etapa

Estabelecer uma paridade visual e jogavel minima do Godot com o prototipo HTML5/Canvas v0.3.4, mantendo Godot como plataforma principal a partir da G2.

Esta etapa melhora a leitura inicial do build Godot Web sem portar sistemas avancados do HTML5.

## Diferenca observada

O HTML5/Canvas v0.3.4 esta congelado como legado/laboratorio, mas ja possui HUD, mapa, composicao visual e identidade dark fantasy mais reconheciveis.

O Godot G1 carregava corretamente no navegador, porem ainda apresentava uma arena simples com player azul, inimigo vermelho e HUD minimo. A G2.1 reduz essa diferenca inicial com desenho procedural simples e feedback de combate mais claro.

## O que foi melhorado

- Player deixou de ser apenas um quadrado azul e passou a ter silhueta simples de necromante.
- Inimigo deixou de ser apenas um quadrado vermelho e passou a ter silhueta simples de criatura.
- Arena recebeu fundo dark fantasy, anel de combate, marcas e runas simples.
- HUD identifica a versao como `Godot G2.1 - baseline visual`.
- HUD mostra vida, mana, vida do inimigo, objetivo e controles.
- Ataque mostra feedback de acerto, erro e dano temporario.
- Inimigo pisca ao receber dano e permanece derrotado visualmente ao chegar a zero de vida.

## Fora do escopo

- Servos.
- Captura.
- Inventario.
- Talentos.
- Save.
- Mapa-mundi.
- Port completo de sistemas HTML5/Canvas.
- Alteracoes em `src/*.js`.

## Como testar

1. Abrir o projeto Godot em `godot/`.
2. Rodar a cena principal `res://scenes/Main.tscn`.
3. Mover o personagem com WASD ou setas.
4. Aproximar-se do inimigo.
5. Atacar com `1` ou clique.
6. Confirmar que o HUD atualiza a vida do inimigo e exibe feedback de acerto/erro.

## Checklist manual

- [ ] Player aparece como necromante simples, nao como quadrado azul puro.
- [ ] Inimigo aparece como criatura simples, nao como quadrado vermelho puro.
- [ ] Arena tem leitura dark fantasy simples.
- [ ] HUD mostra vida, mana e vida do inimigo.
- [ ] HUD mostra `Godot G2.1 - baseline visual`.
- [ ] HUD mostra controles e objetivo.
- [ ] Ataque fora do alcance mostra erro.
- [ ] Ataque no alcance causa dano e atualiza vida do inimigo.
- [ ] Inimigo derrotado mostra status simples.
- [ ] `src/*.js` permanece intocado.
- [ ] HTML5/Canvas legado permanece intocado.

## Comandos de validacao

```bash
npm.cmd run check
git diff --check
git diff --name-only -- src
git ls-files | findstr /i "export_presets.cfg"
godot_console --headless --path godot --quit
godot_console --headless --path godot --export-release "Web" ../docs/play-godot/index.html
```

Se `godot_console` nao estiver disponivel no PATH, usar o executavel local de Godot apenas para validacao/export, sem versionar o binario.
