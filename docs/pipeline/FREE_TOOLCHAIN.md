# Free Professional Toolchain

## Godot

Godot is the engine and source of truth for scenes, gameplay scripts, resources, editor settings and exports. The active project is `godot/project.godot`.

## VS Code

VS Code is used for GDScript editing, docs, search, diffs and lightweight project maintenance.

## Git And GitHub

Git and GitHub manage branches, pull requests, code review, tags, release notes and build history. Work should land through branches and PRs.

## Blender

Blender owns 3D source assets: blockout meshes, props, characters, rigs and animations. Exported game-ready assets belong under `godot/assets/`.

## Krita

Krita owns 2D source art: UI mockups, icons, portraits, texture paint sources and concept paintovers.

## Audacity

Audacity owns sound cleanup, normalization, trimming and export preparation for effects and voice placeholders.

## LMMS

LMMS owns music loops, stingers and soundtrack drafts.

## Kenney

Kenney assets are acceptable as placeholder/base assets when license provenance is recorded. Prefer CC0 and keep third-party sources under `godot/assets/third_party/kenney/`.

## ChatGPT/Codex

ChatGPT/Codex supports architecture review, audits, small corrections, documentation, build/release checklists and regression analysis. It does not replace manual playtesting in Godot.
